Worker
真正执行任务的类,继承自AQS实现Runnable,每个Worker实例封装了一个Thread,同时记录着自己完成的任务总数,他的Run方法就是调用外部的runWorker方法。其中有一个firstTask属性,该属性表示的实例化Worker时传入的任务,可以为null。Worker会不断尝试去任务队列中获取任务,等待任务的worker称之为闲置worker(上面说到的空闲线程)。

shutdown()分析

shutdown的关键就是想办法让worker退出runWorker方法,退出的方式只有一个:getTask返回null。
1.getTask中第一个if判断”线程池状态为>=STOP”或”线程池状态为SHUTDOWN且任务队列为空”,直接返回null,SHUTDOWN已在调用shutdown方法时就被设置,而任务队列是否为空就交由worker线程处理(shutdown方法本身就是这个逻辑:不接收任务,但会处理完任务队列中的现有任务)。
2.getTask中第二个if判断用于处理超时(超时机制一定要开启(timed=true)),如果超时了,直接返回null。
如果线程阻塞在了queue的poll/take方法上,那么说明任务队列一定为空,同时shutdown方法所触发的interruptIdleWorkers一定可以成功调用这个阻塞worker的线程的interrupt方法以让其从queue的阻塞上返回,当其返回后会进入新一轮的循环,新循环中就会成功在1处返回null。

shutdownNow()的”Now”

该方法的调用会直接将线程池状态设置为STOP同时不会在”获取到worker的锁”的前提下中断worker线程,而是直接调用worker的实例方法进行线程中断,当被中断后,worker仍然会在runWorker方法内进入getTask,然后此时在第一个if判断处会不查看任务队列而直接返回(因为是STOP)。
但shutdownNow并不一定能直接停止正在运行任务的worker,如果任务本身不响应中断,那么无法停止该线程,因为Thread.interrupt对于正在运行中的线程仅仅是置interrupt位,所以如果有某些动作要等待线程池的所有线程都退出再执行,那么需要调用awaitTermination方法,这个condition的唤醒在tryTerminate中。
